# 予防アプリ 設計書 v0.1

## 1. コンセプト
予防医学のエビデンスをもとに、**性別・年齢・目的**に合わせた1週間の
**食事／運動／睡眠／休息／瞑想**プランを提案し、毎日の実践を記録・管理するアプリ。

### 不変の方針（コア・フィルタ）
- 検査・専用器材を使わない（クリニック誘導なし）
- 食材は**スーパーで買えるもの**
- サプリは**iHerb等ネットで簡単に買えるもの**のみ
- 運動は**自宅でできる範囲**（自重中心）
- **一人で・簡単に・継続できる**ことが最優先
- **サーバーなし**（1つのWebアプリ、データは端末内 localStorage に保存）

---

## 2. ユーザーフロー
```
[起動]
  └─ プロフィール未設定？
       ├─ YES → オンボーディング（3ステップ）
       │        Step1 性別  → Step2 年齢  → Step3 目的
       │        → プラン自動生成
       └─ NO  → ホーム（今日のプラン＋記録）
```

### オンボーディング入力
| 項目 | 選択肢 |
|---|---|
| 性別 | 男性 / 女性 |
| 年齢 | 数値入力 → 年代に自動マッピング（20代/30代/40代/50代/60代+） |
| 目的 | ①減量 ②筋肉・体力アップ ③活力・疲労回復 ④長期の健康・若さ維持 |

> 将来拡張: 活動量、アレルギー、苦手食材（プロフィールに追加可能な設計にする）

## 3. パーソナライズの考え方（性別 × 年代 × 目的）
コンテンツは「カード」単位で持ち、各カードに**適用タグ**を付ける。
アプリは `{性別, 年代, 目的}` で**フィルタ＋優先度づけ**して提案（ルールベース＝APIキー不要）。

### 年代による調整の軸
| 年代 | 重点 |
|---|---|
| 20代 | 基礎習慣の確立・体力ベース作り |
| 30代 | ホルモン維持・回復重視・血糖/糖化対策 |
| 40代 | 代謝低下対策・柔軟性/関節・骨密度 |
| 50代 | タンパク質増・バランス/転倒予防・睡眠の質 |
| 60代+ | 筋量維持・関節保護・低強度高頻度 |

### 性別による調整の軸
| 性別 | 重点 |
|---|---|
| 男性 | テストステロン維持（下半身トレ・睡眠・亜鉛/マグ）、内臓脂肪 |
| 女性 | 月経周期に合わせた強度・栄養、鉄分、骨密度（カルシウム/ビタミンD）、40-50代は更年期ケア |

### 目的による調整の軸
| 目的 | 食事 | 運動 |
|---|---|---|
| 減量 | 16時間TRE/血糖安定・野菜とタンパク質先 | 有酸素＋自重サーキット |
| 筋肉・体力 | タンパク質＋糖質しっかり | 自重筋トレ漸進・週3 |
| 活力・疲労回復 | 鉄/オメガ3・ミトコンドリア | インターバル速歩・低強度 |
| 長期の健康 | 地中海食・食物繊維・発酵 | 速歩＋筋トレ＋柔軟のバランス |

> ⚠️ 女性向けの詳細データは現在収集中（月経周期・鉄・更年期）。集まり次第カードを追加。

## 4. 画面一覧
1. **オンボーディング**（性別→年齢→目的）
2. **ホーム/ダッシュボード** … 今日のプラン要約＋ワンタップ記録＋達成リング
3. **週間プラン** … 7日分の食事/運動/休息スケジュール
4. **食事** … 1日の献立例・スーパー食材リスト・食べ方ルール
5. **運動** … 自宅メニュー（種目・回数・動画/図）
6. **睡眠・休息** … 就寝ルーティン・サウナ/入浴・休息日
7. **瞑想・呼吸** … 呼吸法・短時間瞑想の手順＋タイマー
8. **サプリ** … iHerbで買える候補（目的別・任意）
9. **記録/履歴** … 体重・睡眠・運動・気分の推移グラフ
10. **設定** … プロフィール編集・データ書き出し/初期化

```jsonc
  "id":"meal-bloodsugar-01",

  "body":"...",
  "applies":{ "sex":["male","female"], "ageBand":["30s","40s","50s"], "goal":["fatloss","longevity"] },
  "priority":3,
  "source":"hii1VzgEcQU / advanced_system",
  "evidence":3                     // ★信頼度 1-3
}]
```

## 6. 技術構成
- **ビルド不要の静的Webアプリ**（`index.html` ＋ JS ＋ CSS ＋ JSON）
- フレームワーク: まずは**Vanilla JS**（依存ゼロ・軽量・保守簡単）。必要なら後でグラフ用に軽いCDNライブラリ
- データ: ユーザーデータは localStorage、知識は同梱JSON
- グラフ: Canvas か 軽量チャートライブラリ（記録の推移）
- PWA対応（任意）: ホーム画面追加・オフライン動作

## 7. 提案エンジン（APIキー不要）
1. プロフィール `{sex, ageBand, goal}` を読む
2. `cards.json` から `applies` がマッチするカードを抽出
3. `priority` × `evidence` でスコア付け→各ドメインの上位を採用
4. 週間テンプレートに割り当て（曜日ごとに運動/休息を配置）
5. 生成結果を `yobou.plan` にキャッシュ（「作り直す」で再生成）

> 将来: APIキーを足せば、記録の傾向を見て文章で個別アドバイスする機能を追加可能。
