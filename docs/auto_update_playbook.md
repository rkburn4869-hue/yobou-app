# 自動更新プレイブック（週2回のroutine用）

このリポジトリ（予防アプリ）を、指定ソースの新着から学んで**週2回**更新するための手順。
スケジュール実行されるエージェントは、以下を厳密に守ること。

## 方針（必ず守るフィルタ）
ユーザーのテーマ＝**検査/専用器材なし・スーパーで買える食材・iHerb等で買えるサプリ・自宅でできる運動・一人で簡単に継続**。
- このテーマに合う回**だけ**採用する。合わない回は採用しない。
- **陰山医師チャンネルは宣伝色が強い**（自院検査・サプリ販売・再生医療・LINE誘導）。
  宣伝・自院誘導・断定的な効能（「打率5割」等の未検証数値）は**載せない**。一般化できる予防医学の要点だけ抽出。
- 推奨は具体的・実行可能に。出典(番組名/動画ID)と信頼度★(1-3)を必ず付ける。
- 医療行為の指示や診断はしない。

## 手順
1. 依存セットアップ: `pip install --user yt-dlp faster-whisper`（無ければ）。
2. `python3 tools/check_new.py` を実行し新着候補を取得。
3. 候補から**テーマに合うものを最大3件**選ぶ。コスト配慮で **YouTube(字幕)優先**、Podcastは強く合致する場合のみ1件まで。
4. 学習（生成物は `data/` に保存＝gitignore対象。コミットしない）:
   - YouTube: `python3 tools/fetch_transcript.py "<url>"`
   - Podcast: `python3 tools/fetch_podcast.py rss <appleId> <name>` → `search` → `grab <name> <番号> --model base.en`
5. 文字起こしを読み、要点を抽出。テーマに合致する実用情報のみを:
   - `app/js/articles.js` に3分コラムとして追加（既存の形式・口調に合わせる。bodyは事実ベース・宣伝除外）。
   - 必要なら `app/js/cards.js`（提案カード）/ `app/js/meals.js`（献立）も更新。
   - `data/knowledge_base.md` があれば追記（無ければスキップ可）。
6. **状態更新**: `tools/processed.json` の `keys` に、今回 check_new で出た候補の**全キー**を追記（採用・不採用どちらも。再出現防止）。
7. 反映:
   - **app/ に変更がある場合** → ブランチ `auto/update-YYYYMMDD` を作成しコミット、`gh pr create` で**PRを作成**（マージはしない＝ユーザー承認制）。
     PR本文に「追加した記事/カードの一覧・各出典・テーマ適合理由・品質上の注意」を箇条書きで書く。
   - **processed.json だけの変更（テーマ該当なし）** → その1ファイルだけ `main` に直接コミット&push（状態管理のみ・レビュー不要）。
8. 絶対に `data/` をコミットしないこと（著作権物）。`git status` で確認。

## 失敗時
- ソース取得やWhisperが失敗しても全体は止めない。処理できたものだけ反映し、PR本文に失敗を明記。
- 新着ゼロ/該当なしなら、processed.json の更新だけ行い終了（PRなし）。

## 補足
- check_new はテーマ判定をしない。採否はタイトル＋本文をエージェントが判断する。
- Podcast文字起こしは重い。時間が厳しければ YouTube のみ処理し、Podリンクはメモして次回回しでもよい。
